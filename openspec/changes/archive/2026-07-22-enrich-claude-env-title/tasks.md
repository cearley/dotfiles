- [x] 1.1 In `home/.chezmoitemplates/claude-environments`, replace the single `_claude_env_set_title` body with helper functions: `_claude_env_name` (sets `REPLY` to the bare env name, empty when `CLAUDE_CONFIG_DIR` is unset) and `_claude_env_git_context` (sets `REPLY` to `<repo> · <branch>`, `<repo>` alone, or empty). Both set `REPLY` rather than echoing so prompt-path callers avoid a subshell fork; every caller declares `local REPLY`. `_claude_env_name` is also reused by `claude-env` and `prompt_claude_env`, which previously each carried their own copy of the suffix-stripping logic.

Use the `${title:+$title · }` guard so a separator is emitted only when a segment precedes it — this makes leading, trailing, and doubled ` · ` structurally impossible rather than something the assembly logic has to remember to avoid.
- [x] 1.4 Wire the zsh `precmd` hook to call the assembly function with the idle job name (`zsh`).
- [x] 1.5 Add a new zsh `preexec` hook (`add-zsh-hook preexec ...`) that calls the assembly function with the first word of the about-to-run command as the job.

## 4. Follow-up (note — not a task of this change)

Surfaced by task 2.6, recorded here so it is not lost: `~/.bash_profile` (chezmoi-managed) does not source `~/.bashrc`, so a **login** bash shell loads none of the Claude environment wiring — not just the title hook, but `claude-env` and every `claude-<env>` launcher. Standard practice is for `.bash_profile` to source `.bashrc`. This predates the current change and affects the whole bash surface, so it belongs in its own proposal rather than being folded in here. Verified under a PTY: `bash -l -i` emits no title sequences, `bash -i` works correctly.
