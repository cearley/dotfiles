## ADDED Requirements

### Requirement: Homebrew Tap Trust Declarations
The `packages.yaml` file SHALL include a `packages.darwin.trusted` list identifying which tap names, formula names, and cask names require `, trusted: true` in the generated Brewfile.

#### Scenario: Trusted set defined at darwin level
- **WHEN** `packages.yaml` is parsed
- **THEN** `packages.darwin.trusted` SHALL exist as a flat list of strings
- **AND** each entry SHALL be either a tap name (`owner/tap`) or a fully-qualified formula/cask name (`owner/tap/formula`)

#### Scenario: Tap-level trust covers all formulae from that tap
- **WHEN** a tap name appears in `packages.darwin.trusted`
- **THEN** the generated Brewfile SHALL output `tap "<name>", trusted: true` for that tap
- **AND** formulae and casks from that tap SHALL NOT require individual entries in the trusted set

#### Scenario: Formula-level trust for unmanaged taps
- **WHEN** a formula is installed from a tap that is not declared as a standalone tap entry
- **THEN** the fully-qualified formula name (`owner/tap/formula`) SHALL appear in `packages.darwin.trusted`
- **AND** the generated Brewfile SHALL output `brew "<owner/tap/formula>", trusted: true`

#### Scenario: Cask-level trust using fully-qualified name
- **WHEN** a cask requires formula-level trust
- **THEN** its entry in the `casks` list SHALL use the fully-qualified name (`owner/tap/cask`)
- **AND** that fully-qualified name SHALL appear in `packages.darwin.trusted`
- **AND** the generated Brewfile SHALL output `cask "<owner/tap/cask>", trusted: true`

#### Scenario: Untrusted entry produces no annotation
- **WHEN** an entry does not appear in `packages.darwin.trusted`
- **THEN** the generated Brewfile SHALL output the entry without `, trusted: true`

### Requirement: Uniform String Format for Package Lists
All entries in `taps`, `brews`, and `casks` lists in `packages.yaml` SHALL be plain strings. Dict-format entries (`{name: ..., trusted: true}`) SHALL NOT be used.

#### Scenario: Taps list contains only strings
- **WHEN** `packages.yaml` is read
- **THEN** every entry in any `taps` list SHALL be a plain YAML string
- **AND** no entry SHALL be a YAML mapping

#### Scenario: Brews list contains only strings
- **WHEN** `packages.yaml` is read
- **THEN** every entry in any `brews` list SHALL be a plain YAML string

#### Scenario: Casks list contains only strings
- **WHEN** `packages.yaml` is read
- **THEN** every entry in any `casks` list SHALL be a plain YAML string

### Requirement: Brewfile Trust Annotation in Generated Output
The Brewfile heredoc generated by `run_onchange_before_darwin-23-install-packages.sh.tmpl` SHALL append `, trusted: true` to any tap, brew, or cask line whose name appears in `packages.darwin.trusted`.

#### Scenario: Trust annotation appended to tap line
- **WHEN** the template renders a tap entry
- **AND** the tap name is in `packages.darwin.trusted`
- **THEN** the output line SHALL be `tap "<name>", trusted: true`

#### Scenario: Trust annotation appended to brew line
- **WHEN** the template renders a brew entry
- **AND** the brew name is in `packages.darwin.trusted`
- **THEN** the output line SHALL be `brew "<name>", trusted: true`

#### Scenario: Trust annotation appended to cask line
- **WHEN** the template renders a cask entry
- **AND** the cask name is in `packages.darwin.trusted`
- **THEN** the output line SHALL be `cask "<name>", trusted: true`

#### Scenario: Pre-tap bash section omits trust
- **WHEN** the pre-tap section runs `brew tap` commands before `brew bundle`
- **THEN** no trust flag SHALL be appended to `brew tap` commands
- **AND** the `brew tap` CLI output SHALL be unaffected by the trusted set

## MODIFIED Requirements

### Requirement: Homebrew Package Types
The system SHALL support all Homebrew package types: taps, brews (formulae), casks, and Mac App Store apps.

#### Scenario: Homebrew tap installation
- **WHEN** a tap is defined in `packages.yaml` under the `taps` key
- **THEN** the tap SHALL be added via `brew tap` before installing packages from that tap

#### Scenario: Formula installation
- **WHEN** a package is listed under `brews`
- **THEN** it SHALL be installed via `brew install <package>`
- **AND** the package name MAY be a fully-qualified `owner/tap/formula` string when formula-level trust is required

#### Scenario: Cask installation
- **WHEN** an application is listed under `casks`
- **THEN** it SHALL be installed via `brew install --cask <package>`
- **AND** the cask name MAY be a fully-qualified `owner/tap/cask` string when formula-level trust is required

#### Scenario: Mac App Store installation
- **WHEN** an app is listed under `mas` with format `"App Name", id: 123456`
- **THEN** it SHALL be installed via `mas install 123456`
